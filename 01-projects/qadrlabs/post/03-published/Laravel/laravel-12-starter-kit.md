## Apa Itu Laravel 12 Starter Kit?{#apa-itu-laravel-12-starter-kit}  
Sebelum kita masuk ke detail teknis, ada baiknya kita memahami terlebih dahulu apa itu Laravel 12 Starter Kit. Secara sederhana, *starter kit* adalah template aplikasi yang sudah dikonfigurasi sebelumnya, lengkap dengan rute, kontroler, dan tampilan yang dibutuhkan untuk mendaftarkan dan mengotentikasi pengguna aplikasi Anda.  

### Definisi dan Manfaat  
Bayangkan Anda sedang membangun rumah. Daripada memulai dari nol dengan membeli bahan bangunan, menggambar desain, dan membangun fondasi, Anda mendapatkan rumah prefabrikasi yang sudah hampir selesai. Anda hanya perlu menambahkan sentuhan pribadi untuk membuatnya sesuai dengan keinginan Anda. Nah, itulah yang dilakukan oleh Laravel 12 Starter Kit.  

Dengan menggunakan *starter kit*, Anda dapat langsung fokus pada fitur inti aplikasi Anda tanpa harus khawatir tentang konfigurasi dasar seperti autentikasi pengguna atau manajemen rute. Ini tidak hanya menghemat waktu, tetapi juga memastikan bahwa Anda memulai dengan struktur yang sudah dioptimalkan.  

### Mengapa Harus Menggunakan Starter Kit?  
Meskipun Anda bebas membangun aplikasi Laravel dari awal, menggunakan *starter kit* memiliki beberapa keuntungan signifikan. Pertama, Anda mendapatkan akses ke alat-alat modern seperti React, Vue, dan Livewire, yang semuanya telah diintegrasikan dengan Laravel. Kedua, Anda dapat memanfaatkan teknologi terbaru seperti TypeScript, Tailwind CSS, dan Vite untuk meningkatkan performa aplikasi Anda.  

Namun, yang paling penting adalah efisiensi. Dengan *starter kit*, Anda dapat memotong waktu pengembangan hingga separuhnya, sehingga Anda bisa lebih cepat merilis produk ke pasar.  



## React Starter Kit: Kekuatan React dan Laravel{#react-starter-kit}  
Salah satu opsi *starter kit* yang paling populer di Laravel 12 adalah React Starter Kit. Tapi apa yang membuatnya begitu istimewa?  

### Fitur Utama  
React Starter Kit menawarkan kombinasi sempurna antara kekuatan frontend React dan backend Laravel. Dengan integrasi Inertia.js, Anda dapat membangun aplikasi single-page modern menggunakan routing server-side tradisional. Ini memungkinkan Anda menikmati fleksibilitas React tanpa harus meninggalkan produktivitas Laravel.  

### Integrasi dengan Inertia.js  
Inertia.js adalah alat yang memungkinkan Anda membangun aplikasi React tanpa harus menulis API manual. Semua data dikirim langsung dari backend Laravel ke frontend React melalui server-side rendering. Ini tidak hanya mempercepat pengembangan, tetapi juga meningkatkan performa aplikasi Anda.  

### Integrasi dengan Inertia.js (Lanjutan)  
Dengan Inertia.js, Anda tidak perlu lagi khawatir tentang konfigurasi API yang rumit atau manajemen state secara manual. Semua data yang dibutuhkan oleh frontend React dikirim langsung dari kontroler Laravel melalui server-side rendering. Ini memungkinkan Anda untuk fokus pada logika bisnis aplikasi tanpa harus mengorbankan kecepatan atau skalabilitas.  

Selain itu, React Starter Kit juga dilengkapi dengan TypeScript, Tailwind CSS, dan library komponen `shadcn/ui`. TypeScript memberikan keamanan tipe yang lebih baik, sehingga mengurangi risiko kesalahan saat pengembangan. Sementara itu, Tailwind CSS memungkinkan Anda membuat desain yang responsif dan modern dengan cepat, sementara `shadcn/ui` menyediakan komponen UI siap pakai yang dapat digunakan langsung dalam proyek Anda.  

Jadi, jika Anda mencari cara untuk membangun aplikasi modern dengan performa tinggi, React Starter Kit adalah pilihan yang tepat.  



## Vue Starter Kit: Fleksibilitas Vue dan Laravel{#vue-starter-kit}  
Bagi Anda yang lebih nyaman menggunakan Vue.js, Laravel 12 juga menawarkan Vue Starter Kit. Mirip dengan React Starter Kit, Vue Starter Kit dirancang untuk memberikan pengalaman pengembangan yang mulus dengan integrasi Inertia.js.  

### Penggunaan Composition API  
Salah satu fitur utama Vue Starter Kit adalah dukungan untuk Vue Composition API. Composition API adalah pendekatan modern dalam pengembangan Vue yang memungkinkan Anda mengorganisasi logika komponen dengan lebih baik. Dibandingkan dengan Options API tradisional, Composition API memberikan fleksibilitas yang lebih besar, terutama untuk aplikasi skala besar.  

Dengan Vue Starter Kit, Anda dapat langsung memanfaatkan Composition API untuk membangun komponen yang modular dan mudah dikelola. Ini sangat berguna jika Anda ingin membangun aplikasi dengan struktur kode yang bersih dan terorganisir.  

### Keunggulan TypeScript dan Tailwind CSS  
Seperti halnya React Starter Kit, Vue Starter Kit juga mendukung TypeScript dan Tailwind CSS. TypeScript membantu Anda menghindari bug tipe umum, sementara Tailwind CSS memungkinkan Anda membuat desain yang indah tanpa harus menulis CSS kustom yang rumit.  

Selain itu, Vue Starter Kit menggunakan library komponen `shadcn-vue`, yang merupakan versi Vue dari `shadcn/ui`. Library ini menyediakan komponen UI yang siap pakai, sehingga Anda dapat langsung menggunakannya dalam proyek Anda tanpa harus memulai dari awal.  

Dengan semua fitur ini, Vue Starter Kit adalah pilihan ideal bagi developer yang ingin memanfaatkan fleksibilitas Vue.js dengan backend Laravel yang andal.  



## Livewire Starter Kit: Bangun UI Dinamis dengan PHP{#livewire-starter-kit}  
Jika Anda lebih suka menggunakan PHP daripada JavaScript untuk membangun UI dinamis, Livewire Starter Kit adalah jawabannya. Livewire adalah alat yang memungkinkan Anda membangun antarmuka pengguna yang interaktif hanya dengan PHP, tanpa perlu banyak JavaScript.  

### Mengapa Memilih Livewire?  
Livewire sangat cocok untuk tim yang lebih nyaman menggunakan Blade templates dan ingin menghindari kerumitan framework JavaScript seperti React atau Vue. Dengan Livewire, Anda dapat membangun UI yang reaktif dan dinamis hanya dengan PHP, sehingga mengurangi kompleksitas proyek Anda.  

Salah satu keuntungan utama Livewire adalah kemudahan penggunaannya. Anda tidak perlu belajar bahasa baru atau alat baru—semuanya dapat dilakukan dengan PHP yang sudah Anda kuasai. Ini membuat Livewire menjadi pilihan yang sangat efisien untuk proyek-proyek yang membutuhkan UI dinamis tetapi tidak ingin mengorbankan produktivitas.  

### Komponen Flux UI dan Laravel Volt  
Livewire Starter Kit dilengkapi dengan dua alat penting: Flux UI dan Laravel Volt. Flux UI adalah library komponen UI berbasis Tailwind CSS yang dirancang khusus untuk Livewire. Dengan Flux UI, Anda dapat dengan cepat membangun antarmuka pengguna yang modern dan responsif tanpa harus menulis CSS kustom.  

Sementara itu, Laravel Volt adalah alat baru yang memungkinkan Anda menulis logika Livewire langsung di file Blade. Ini menghilangkan kebutuhan untuk membuat file kontroler terpisah, sehingga membuat kode Anda lebih ringkas dan mudah dikelola.  

Dengan kombinasi Livewire, Flux UI, dan Laravel Volt, Anda dapat membangun aplikasi yang dinamis dan responsif hanya dengan PHP. Jadi, jika Anda mencari cara untuk mempercepat pengembangan tanpa harus belajar teknologi baru, Livewire Starter Kit adalah pilihan yang sempurna.  

## Cara Memulai dengan Laravel 12 Starter Kit{#cara-memulai-dengan-laravel-12-starter-kit}  
Sekarang, mari kita bahas bagaimana cara memulai dengan Laravel 12 Starter Kit. Proses ini cukup sederhana, tetapi memerlukan beberapa langkah penting.  

### Instalasi PHP dan Laravel CLI  
Langkah pertama adalah memastikan bahwa Anda telah menginstal PHP dan Laravel CLI di sistem Anda. Jika Anda belum melakukannya, Anda dapat menginstal Laravel CLI melalui Composer dengan command berikut:  
```bash
composer global require laravel/installer
```  
Setelah instalasi selesai, Anda dapat memverifikasi bahwa Laravel CLI telah terpasang dengan running:  
```bash
laravel --version
```  

### Membuat Aplikasi Baru  
Setelah Laravel CLI terpasang, Anda dapat membuat aplikasi baru dengan running command berikut:  
```bash
laravel new my-app
```  
Saat running command ini, Laravel CLI akan meminta Anda untuk memilih *starter kit* yang diinginkan. Anda dapat memilih antara React, Vue, atau Livewire, tergantung pada kebutuhan proyek Anda.

### Run Project  
Setelah aplikasi berhasil dibuat dan asset front dibuild, kita bisa coba untuk running project laravel. Kita masuk terlebih dahulu ke direktori project:  
```bash
cd my-app
```  
Lalu, jalankan server pengembangan Laravel dengan command:  
```bash
composer run dev
```  
Setelah server berjalan, aplikasi Anda akan dapat diakses di browser melalui `http://localhost:8000`.

### Kendala yang ditemui {#kendala-yang-ditemui}
Ketika mencoba membuat aplikasi baru menggunakan `laravel` command.
```
laravel new my-app
```
Starter kit baru tidak ditampilkan ketika prompt muncul:
```

   _                               _
  | |                             | |
  | |     __ _ _ __ __ ___   _____| |
  | |    / _` | '__/ _` \ \ / / _ \ |
  | |___| (_| | | | (_| |\ V /  __/ |
  |______\__,_|_|  \__,_| \_/ \___|_|


 Would you like to install a starter kit? [No starter kit]:
  [none     ] No starter kit
  [breeze   ] Laravel Breeze
  [jetstream] Laravel Jetstream
```
Opsi pada prompt masih menampilkan opsi dari laravel installer versi lama, yaitu no starter kit, laravel breeze dan laravel jetstream. 


Solusi untuk memperbaikinya adalah dengan cara *update versi laravel installer*. Untuk update versi laravel installer, run command berikut ini.
```
composer global update laravel/installer
```
Tunggu sampai proses update selesai.

Selanjutnya kita cek versi laravel installer yang terinstall.
```
laravel --version
```

Selanjutnya kita bisa langsung coba kembali untuk membuat project baru.
```
laravel new my-app
```
