# Continue Learning: Learn Laravel Beyond the Basics

You built Catatku from an empty folder to a working, private journal application, and you understand why each piece is there. That is a real foundation. The natural question now is: what comes next? The answer is a follow-up course built directly on the app you just made.

**Learn Laravel: Beyond the Basics** picks up exactly where this book ends. It uses the same Catatku project, the same `Entry` model, the same `x-layout` and `x-entry-card` components, and the same authentication system as the starting point, then extends them with the features every production Laravel application eventually needs.

## What the Sequel Covers {#what-the-sequel-covers}

The course adds one feature or concept at a time, the same way this book did. By the end, Catatku grows from a personal journal into a production-grade application with:

- **Database relationships** - comments on entries (one-to-many) and tags (many-to-many) with pivot tables.
- **Advanced Eloquent** - query scopes, accessors and mutators, eager loading, soft deletes, and pagination.
- **Authorization** - Gates and Policies so the ownership checks you wrote by hand become clean, centralized rules, plus custom middleware and rate limiting.
- **File uploads** - cover images for entries using the Storage facade.
- **Email** - welcome emails and comment notifications with Mailables.
- **A REST API** - JSON endpoints, API resources, and token authentication with Sanctum so a mobile app or SPA can talk to Catatku.
- **Automated testing** - feature and unit tests with Pest, factories, and database testing.
- **Background processing** - queues, jobs, events, and listeners for work that should not block a request.
- **Frontend tooling** - more advanced Blade components and a real Vite plus Tailwind CSS build, replacing the CDN setup from this book.
- **Deployment to production** - taking Catatku live on a real server, the proper way.

## Why It Is the Right Next Step {#why-the-right-next-step}

Everything in the sequel reuses the mental model you already have. You will not start over with a new project or relearn the basics. The request flow through MVC, the validate-then-save pattern, and the ownership rules you practiced here are the exact foundation the advanced topics build on. That continuity is the whole point: you keep deepening one real application instead of collecting disconnected demos.

If you finished this book and want to keep going, this is where to go.
