# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

This is an Obsidian vault organized using the PARA method (Projects, Areas, Resources, Archives). It's a personal knowledge management system where notes are stored as Markdown files.

## Architecture

The vault follows the PARA organizational structure:

- **00-inbox/**: Temporary holding area for new, unprocessed notes and ideas
- **01-projects/**: Short-term efforts with specific goals and deadlines
- **02-areas/**: Long-term responsibilities and areas of focus without specific end dates
- **03-resources/**: Topics of ongoing interest, reference materials, and learning resources
- **04-archives/**: Completed or inactive items from other folders

## File Organization

- All content files are Markdown (.md) format
- Notes should be placed in the appropriate PARA folder based on their nature:
  - Active work with deadlines → `01-projects/`
  - Ongoing responsibilities → `02-areas/`
  - Reference materials → `03-resources/`
  - Quick capture before organization → `00-inbox/`
  - Completed items → `04-archives/`

## Working with Notes

This vault uses Obsidian features:
- **Wikilinks**: Use `[[Note Name]]` for internal linking between notes
- **Tags**: Use `#tag` for categorization
- **Frontmatter**: YAML metadata at the top of notes (between `---` delimiters)
- **Backlinks**: Notes can reference each other bidirectionally

When creating or modifying notes, maintain consistency with Obsidian's markdown flavor and linking conventions.
