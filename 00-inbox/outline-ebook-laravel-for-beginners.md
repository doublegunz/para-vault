# Outline Ebook Laravel for Beginners

Catatan ini merangkum ide struktur ebook untuk materi course:

`01-projects/qadrlabs/courses/learn-laravel-for-beginners`

Prinsip utamanya: isi materi utama tetap mengikuti urutan course. Pembeda ebook bukan dari mengubah alur belajar, tetapi dari tambahan pengalaman membaca seperti halaman legal, panduan penggunaan, checklist, troubleshooting, cheat sheet, bonus, dan informasi update.

## Arah Ebook

Judul kerja:

**Belajar Laravel untuk Pemula: Membangun Aplikasi Catatku dari Nol**

Alternatif judul:

- **Learn Laravel for Beginners: Build Catatku from Scratch**
- **Laravel untuk Pemula: Dari Project Kosong sampai Aplikasi Catatan**
- **Belajar Laravel 13 untuk Pemula dengan Project Catatku**

Target ebook:

- Pemula yang sudah tahu dasar PHP, tetapi belum pernah memakai framework.
- Pembaca yang pernah mencoba Laravel, tetapi masih bingung hubungan route, view, controller, model, database, dan autentikasi.
- Pembaca yang ingin membangun project utuh, bukan hanya latihan potongan kode.

## Catatan HAKI dan Lisensi

Ebook perlu punya halaman legal yang jelas. Bagian ini tidak perlu terlalu panjang, tetapi harus tegas.

Contoh isi halaman hak cipta:

```text
Copyright © 2026 QadrLabs. Seluruh hak cipta dilindungi undang-undang.

Ebook ini hanya boleh digunakan oleh pembeli atau peserta yang memiliki akses resmi. Dilarang memperbanyak, menjual ulang, mengunggah, membagikan, atau mendistribusikan sebagian maupun seluruh isi ebook ini tanpa izin tertulis dari pemegang hak cipta.
```

Tambahkan juga bagian penggunaan kode:

```text
Kode contoh dalam ebook ini boleh digunakan untuk belajar, latihan pribadi, dan pengembangan project pribadi. Kode contoh tidak boleh dijual ulang sebagai template, course, ebook, atau produk turunan lain tanpa izin tertulis dari QadrLabs.
```

Tambahkan disclaimer merek:

```text
Laravel, PHP, Composer, Visual Studio Code, Laragon, MySQL, dan produk lain yang disebutkan dalam ebook ini adalah merek atau produk milik pemegang hak masing-masing. Ebook ini tidak berafiliasi secara resmi dengan pemilik merek tersebut kecuali dinyatakan lain.
```

Catatan praktis:

- Hak cipta pada dasarnya melekat otomatis setelah karya diwujudkan.
- Pencatatan hak cipta ke DJKI dapat dipertimbangkan untuk memperkuat bukti kepemilikan.
- Simpan bukti draft, file sumber, tanggal publikasi, invoice penjualan, dan riwayat revisi.
- Jika ebook akan dijual luas, sebaiknya konsultasi legal untuk teks lisensi final.

## Struktur Terbaik

Struktur akhir yang paling rapi:

```text
Front Matter
Materi Course 12 Lesson
Back Matter
Bonus
Legal
```

Namun secara layout buku, legal tetap sebaiknya muncul di awal. Maksud struktur di atas adalah:

- Bagian depan membantu pembaca memahami konteks.
- Bagian tengah mempertahankan alur course.
- Bagian belakang memperkuat nilai ebook dengan checklist, troubleshooting, cheat sheet, dan roadmap.

## Pembeda Ebook

Pembeda ebook dari course atau ebook Laravel lain:

- Project utuh: Catatku.
- Urutan belajar tetap natural seperti course.
- Tidak memakai starter kit atau scaffolding autentikasi.
- Pembaca menulis route, view, controller, model, CRUD, dan auth sendiri.
- Ada checklist akhir modul.
- Ada troubleshooting khusus pemula.
- Ada cheat sheet.
- Ada source code final.
- Ada roadmap lanjutan setelah project selesai.
- Ada update/revisi versi yang jelas.

## Catatan Produksi

Sebelum dijual:

- Pastikan semua command masih cocok dengan Laravel 13.
- Pastikan versi PHP minimum disebutkan konsisten.
- Pastikan screenshot sesuai dengan versi materi.
- Siapkan source code final sebagai file ZIP atau repository private/public.
- Siapkan halaman changelog.
- Siapkan halaman download dan update.
- Siapkan lisensi yang jelas untuk PDF dan source code.
- Buat versi PDF yang nyaman dibaca di desktop dan tablet.
- Pertimbangkan EPUB hanya jika layout kode tetap rapi.

## Rekomendasi Final

Untuk QadrLabs.com, gunakan outline berbahasa Indonesia atau campuran sesuai target audiens QadrLabs. Karena materi course saat ini berbahasa Inggris, tentukan sejak awal apakah ebook akan:

- Tetap berbahasa Inggris mengikuti course.
- Diterjemahkan ke bahasa Indonesia.
- Dibuat bilingual secara terbatas, misalnya istilah teknis tetap Inggris tetapi penjelasan Indonesia.

Jika ebook dijual di Gumroad untuk audiens global, lebih aman mempertahankan bahasa Inggris dan memakai struktur Gumroad yang mandiri.

Jika ebook dijual di QadrLabs.com untuk audiens Indonesia, versi Indonesia bisa menjadi pembeda yang kuat, selama istilah Laravel tetap konsisten.
